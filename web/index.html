// Features

Your Habits, Under Control.

Six intelligent systems working in concert to give you complete visibility and control over your digital day.

📊

Advanced Activity Logging

Tracks active applications, window titles, URLs visited, session durations, keystrokes, and mouse clicks — every moment of your desktop day, organized into searchable CSV logs.

Intelligent Idle Detection

Smart Subtraction

Auto-Block

Smart Detection

// How it Works

Four Layers of Awareness

Stasis operates as a layered system — each tier continuously feeds the next.

01

Activity Monitoring

Lightweight system hooks track the focused app, keystroke intensity, clicks, window titles, and URLs — continuously.

02

Idle & Media States

No input for 2 min triggers Idle Mode, subtracting that time from stats. Active media overrides idle so data stays accurate.

03

Local Data Storage

All telemetry lives in local CSV files and databases on your machine. Nothing leaves unless you explicitly request it via Telegram.

04

Secure Telegram Bridge

A secure long-polling connection links Stasis to the Telegram API — your private bot is a command interface from anywhere in the world.

// Architecture

Full-Stack. Native. Fast.

Stasis combines a Python telemetry engine with a modern Tauri + React frontend, packaged by a single automated build script.

🐍

Python Backend Engine

The core telemetry layer — monitors active windows, logs keystrokes and mouse activity, detects idle states, handles media exceptions, watches the file system, and manages the Telegram long-polling bot. Compiled to a headless .exe via PyInstaller.

Python 3.8+PyInstallerTelegram Bot APIwatchdogpywin32
🆕

React / Vite Frontend

A responsive UI built in React + Vite, wrapped in a Rust-powered Tauri shell. Native Windows feel with web-grade rendering — no Electron overhead, just a tight native binary with a polished interface.

ReactViteTauriRustNode.js
📦

NSIS / Inno Installer

The automated build.ps1 PowerShell script compiles both the Python backend and the Tauri frontend, then bundles them into a polished NSIS Windows installer — one .exe that handles everything cleanly.

NSISInno SetupPowerShellbuild.ps1
📊

Local-First Data Layer

All telemetry — activity logs, file system events, session durations — lives exclusively on your machine as CSV files and local databases. Data is only transmitted when you explicitly request it via a /log command.

CSV LogsSQLiteLocal-onlyNo Cloud

Build Steps

1

Clone the Repository

git clone https://github.com/arshsisodiya/Stasis.git then cd Stasis

2

Configure Credentials

Rename config.template.json to config.json and insert your Telegram Bot Token and Chat ID.

3

Run the Build Script

Open PowerShell at project root and run .\ build.ps1 — it compiles the Python backend, builds the Tauri frontend, and outputs a packaged installer.

4

Find Your Installer

Output lands at frontend\src-tauri\target\release\bundle\ — run the .exe installer and you're done.

Backend-only dev mode: Skip the full build — run python -m venv venv → venv\Scripts\activate → pip install -r requirements.txt → python src/main.py
